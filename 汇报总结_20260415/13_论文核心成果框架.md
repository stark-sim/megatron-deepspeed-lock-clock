## 一、发现：Baseline 动态频率的隐性性能损失

### 1.1 问题定义

在大规模分布式深度学习训练中，GPU 默认采用动态 boost 频率策略（baseline）。名义上 GPU 运行于最高频率（如 RTX 4080 SUPER 的 2505 MHz、V100 的 1380 MHz），但实际训练过程中：

1. **计算负载持续高功耗 → GPU 温度飙升**
2. **温度超过热保护阈值 → 驱动自动降频（thermal throttling）**
3. **多卡之间降频节奏不同步 → 计算进度不一致 → 产生额外等待开销**
4. **最终效果：名义高频的实际有效吞吐反而低于稳定中频**

### 1.2 实验证据

| 平台 | 配置 | Baseline | Static 中频 | 现象 |
|------|------|----------|-------------|------|
| 4080S | Qwen2.5-7B, 8-GPU | 2505 MHz, 243.0s | 1650 MHz, 246.9s | 时间几乎持平，功率降 28% |
| 4080S | LLaMA-7B, 8-GPU | 2505 MHz, 281.0s | **1800 MHz, 276.1s** | **中频反而更快** |
| V100 | Qwen2.5-7B, 16-GPU | 1380 MHz, 724.6s | **1260 MHz, 717.6s** | **中频反而更快** |
| V100 | LLaMA-7B, 16-GPU | 1380 MHz, 659.0s | 1260 MHz, 620.5s | 中频反而更快 |

**关键洞察**："中频比高频快"不是计算本身的奇迹，而是**baseline 动态频率不稳定性的暴露**。

### 1.3 物理机制拆解

```
Baseline 运行链路：
  名义 2505 MHz → 高功耗 → 温度 > 83°C → 驱动自动降频至 2100-2300 MHz
                                   ↓
                    8 张卡各自独立降频，节奏不一致
                                   ↓
                    NCCL all-reduce 等待最慢的那张卡
                                   ↓
                    有效 step time = 计算时间 + 同步等待 + 降频恢复延迟
                                   ↓
                    最终总时间 ≈ 甚至 > 稳定 1800 MHz 静态运行

Static 1800 MHz 链路：
  固定 1800 MHz → 中等功耗 → 温度稳定在 65-70°C → 从不降频
                                   ↓
                    8 张卡频率完全一致，计算进度同步
                                   ↓
                    NCCL all-reduce 无额外等待
                                   ↓
                    有效 step time = 纯计算时间 + 通信时间（可预测）
```

## 二、方法：静态锁频 + 预测驱动频点优选

### 2.1 静态锁频消除不确定性

通过 `nvidia-smi -lgc` 将 GPU 锁定在目标频率，从根本上消除：
- 热节流导致的频率抖动
- 多卡之间的频率不同步
- 不可预测的同步等待开销

### 2.2 预测模型减少扫频成本

静态锁频的代价是**需要预先知道哪个频点最优**。全频段盲扫（如 1005–2505 MHz 每 15 MHz 一个点）成本极高。

本项目提出的解决方案：
1. **少量扫频实验**（9 个频点）建立物理模型
2. **模型外推**预测全频段的时间-功率-能耗曲线
3. **Pareto 筛选**锁定"时间不拖慢 + 能耗显著降低"的最优区间
4. **在最优区间内密集验证**（3-5 个频点），确认模型预测

## 三、模型：三项瓶颈耦合的性能-能耗预测

### 3.1 时间模型

训练 step 时间由三项瓶颈通过加权调和平均耦合决定：

```
T(f) = tokens_per_step / throughput(f)

throughput(f) = HarmonicBlend(L_compute(f), L_memory, L_comm) × θ_thermal(f)

其中：
  L_compute(f) ∝ f          (计算瓶颈随频率线性提升)
  L_memory   = const        (显存带宽与频率无关)
  L_comm     = const        (通信带宽与频率无关)
  θ_thermal(f) = 1          (静态运行无热节流)
```

简化拟合形式：
```
T_static(f) = a · (f_max / f)^b + c

- a: 最大频率下的纯计算时间
- b: 频率敏感度（b≈1 为计算主导，b<<1 为通信主导）
- c: 与频率无关的通信等待时间
```

### 3.2 功率模型

```
P(f) = P_static + P_dynamic · (f / f_max)^exp

- P_static: 静态功耗（内存、系统开销）
- P_dynamic: 动态功耗范围
- exp: 功耗-频率敏感指数（4080S 约 0.5，V100 约 5.0）
```

### 3.3 能效模型

```
E(f) = P(f) × T(f)                     (总能耗)
tokens/J = total_tokens / E(f)          (能效指标)
```

## 四、验证：多平台、多模型、多拓扑

### 4.1 验证矩阵

| 平台 | GPU | 拓扑 | 模型 | 最优频点 | 能耗降幅 | 时间变化 |
|------|-----|------|------|---------|---------|---------|
| RTX 4080S | 8× SUPER | TP2/PP2/DP2 | Qwen2.5-7B | 1650 MHz | -27.1% | +1.6% |
| RTX 4080S | 8× SUPER | TP2/PP2/DP2 | LLaMA-7B | 1800 MHz | -28.1% | **-1.7%** |
| V100 | 16× SXM3 | TP1/PP2/DP8 | Qwen2.5-7B | 1260 MHz | -25.8% | **-1.0%** |
| V100 | 16× SXM3 | TP1/PP2/DP8 | LLaMA-7B | 1260 MHz | -28.0% | -5.8% |

### 4.2 模型验证精度

| 数据集 | Time MAPE | Power MAPE |
|--------|-----------|------------|
| 4080S Qwen (10 pts) | 1.08% | 1.96% |
| 4080S LLaMA (4 pts) | 2.16% | 0.06% |
| V100 LLaMA (2 pts cal → full curve) | <5% | <1% |

### 4.3 关键结论

1. **静态锁频在跑满 GPU 的多卡场景下效果显著**：能耗降低 20%–30%，时间损失 <3%
2. **"中频反超高频"是 baseline 不稳定性的直接证据**：不是计算加速，而是消除了动态降频的隐性损失
3. **预测模型仅需 9 个扫频点即可建立**：将验证成本从"全频段盲扫"压缩到"区间内 3-5 点确认"
4. **模型参数跨模型可迁移**：同一硬件上功率模型参数不变，仅需调整计算/通信系数

## 六、与现有工作的区别

| 维度 | 传统方法 | 本项目 |
|------|---------|--------|
| 频率策略 | 动态 boost（默认） | **静态锁定（消除不确定性）** |
| 节能方式 | 盲目降频或经验固定 | **预测驱动，Pareto 最优** |
| 性能保障 | 无 | **时间不拖慢作为硬约束** |
| 多卡同步 | 忽略 | **显式利用同步收益** |
| 物理模型 | 单一瓶颈 | **三项耦合 + 热节流分析** |
