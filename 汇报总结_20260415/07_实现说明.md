# 实现说明

## 目标

这一版整理的目标是补齐一套适合汇报和演示的材料包，清楚展示：

1. 在这个项目里什么叫 `baseline`
2. 如何启动一个匹配的 `static` 对照实验
3. 如何公平比较 Zeus 指标
4. 预测层在整个流程中的准确定位

## 范围

这一版故意不直接使用完整 7B 路径，而是选择更适合演示的小模型 preset。

当前默认选择：

- `Qwen2.5-style 1.5B` 架构
- 单机优先
- 较短训练步数
- `DISABLE_CHECKPOINT=1`

这样做的原因是：

- 更容易讲清楚
- 更适合 live demo
- 仍然复用项目已有的 canonical launcher
- 重点落在流程展示，而不是长时间训练

## 交付内容

### 1. 汇报文档

统一放在本目录下，包括：

- 实验口径
- PPT 提纲
- 页面文案
- 演示流程
- 证据清单
- 讲稿

### 2. demo 启动脚本

在 `脚本/运行1p5b演示.sh` 中提供一个小模型 preset，对外暴露 baseline 和 static 两种模式，但底层继续复用 `scripts/run_experiment.sh`。

### 3. sweep 脚本

在 `脚本/批量运行1p5b对比.sh` 中顺序执行：

- 一个 baseline
- 多个 static 频点
- 最后自动调用 Zeus 对比脚本

### 4. Zeus 对比脚本

在 `脚本/对比Zeus结果.py` 中直接读取 `run.json` 中的 Zeus 字段，输出一张适合贴进 PPT 的 Markdown 表格。

## 非目标

- 不引入新的训练逻辑
- 不引入新的预测器逻辑
- 不自动生成 `.pptx`
- 不强行把 `1.5B` 宣称为唯一 demo 规模

## 关键假设

- 当前 repo 已经有稳定的 launcher 和 run artifact 流程
- tokenizer 可以继续复用现有的解析逻辑
- 对于汇报和 live demo 来说，`~1B` 量级 demo 比 7B 更合适
- `1.5B` 比 `0.5B` 更接近当前 Qwen-style 实验叙事，同时仍然足够轻量
