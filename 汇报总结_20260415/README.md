# 汇报总结材料包

这个目录集中存放当前项目里用于汇报、总结、PPT 展示和 live demo 的材料。

本次整理后的原则是：

1. 不再把汇报材料放在 `.context/`。
2. 汇报文档、讲稿、证据清单、演示流程、辅助脚本都放在这一个目录下。
3. 内容统一使用中文。

## 目录结构

| 文件 | 用途 |
|------|------|
| `01_实验口径与主线.md` | baseline / static 的公平对比口径，以及汇报主线 |
| `02_PPT提纲.md` | 适合直接扩写成 PPT 的页结构 |
| `03_PPT页面文案.md` | 可以直接搬进 PPT 的中文文案 |
| `04_演示流程.md` | terminal demo 展示顺序与命令 |
| `05_证据清单.md` | 当前可引用的核心数字、证据等级和表述边界 |
| `06_讲稿.md` | 30 秒、90 秒和摘要级别的讲稿 |
| `07_实现说明.md` | 这套汇报包和 demo 脚本的设计说明 |
| `08_完整PPT与汇报材料.md` | **16 页正式汇报 PPT 的内容说明与使用指南** |
| `GPU能效优化_20%+功耗节省方案.pdf` | **中文版 PPT 文件（主要版本）** |
| `main_zh.tex` | 中文版 LaTeX 源文件 |
| `Makefile` | 编译脚本（使用 XeLaTeX） |
| `脚本/` | 与汇报直接配套的 demo 启动脚本和 Zeus 对比脚本 |

## 最推荐的使用顺序

### 准备阶段
1. 先读 `01_实验口径与主线.md`
2. 再读 `02_PPT提纲.md`

### 制作 PPT
3. 做页内容时直接使用 `03_PPT页面文案.md`
4. 完整 PPT 参考 `08_完整PPT与汇报材料.md` 和配套 PDF

### 现场演示
5. 需要现场演示时按 `04_演示流程.md` 执行
6. 写结论或表格时查 `05_证据清单.md`

## 当前建议主表述

- `Baseline` 是 Megatron-DeepSpeed 的原始默认运行方式。
- `Static` 是在相同 workload 与相同拓扑下，只改变 GPU 时钟策略的公平对照实验。
- 训练时间、平均功率、总能耗统一由 Zeus 采集。
- 合适的固定频点可以在不明显拉长训练时间的前提下，降低约 `20%` 量级的平均功率，部分拓扑更高。
- 预测层的作用是帮助用户更快找到值得验证的频点，而不是替代真实实验。

## 配套脚本

- `脚本/运行1p5b演示.sh`
  - 运行一个小模型 baseline 或 static demo
- `脚本/批量运行1p5b对比.sh`
  - 顺序运行 baseline 和多个 static 点，并自动整理结果
- `脚本/对比Zeus结果.py`
  - 从 `run.json` 中提取 Zeus 指标并生成 Markdown 对比表
