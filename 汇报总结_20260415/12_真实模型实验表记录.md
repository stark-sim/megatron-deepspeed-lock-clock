- 本页记录 2026-04-19 至 2026-04-20 在 `sd-1 + sd-2` Ethernet 双机环境完成的首组 **真实 checkpoint + 真实架构 + 当前项目训练数据前缀** 的同拓扑 `baseline/static` 对照，当前已覆盖 `1005 / 1200 / 1395 / 1500 / 1650 / 1800 / 1950 / 2100 / 2250 MHz` 九个 static 点。

## 这组表格支持的表述

- `baseline` 与九个 `static` 的工作负载是公平对照，主变量只有是否锁频以及锁到哪个频点。
- 在真实 `Qwen2.5-7B-Instruct` checkpoint 与 Ethernet 双机拓扑下，九个固定频点都能把总能耗压到 baseline 的约 `74%~78%`。
- `1200 MHz` 基本支配 `1005 MHz`：总能耗几乎相同，但总时长更短。
- `1500 MHz` 已同时优于 `1395 MHz` 的运行时间与总能耗，说明继续上探高频是有价值的。
- `1950 MHz` 已给出当前最快运行时间，但它相对 `1650 MHz` 的收益主要只剩不到 `1s` 的时长差，而能耗与 `tokens/J` 已经回退。
- `2100 MHz` 与 `2250 MHz` 进一步说明：高频端仍然稳定，但 sweet spot 没有继续上移，`2250` 与 `1650` 几乎同速却更耗电。
- 当前九点里更适合作为主展示点的仍是 `1650 MHz`：它把时间代价压到 `+3.84%`，同时把总能耗进一步降到 `-25.99%`。
- 若要继续拔高证据等级，需要把“真实模型”与“真实数据集”分开表述：当前这组结果已经足够支撑前者，但对后者仍需确认非占位语料后再补跑。
