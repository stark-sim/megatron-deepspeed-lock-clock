## static 比 baseline 多的处理

| 项目 | baseline | static | 备注 |
| --- | --- | --- | --- |
| `EXPERIMENT_MODE` | `baseline` | `static` | 模式位不同 |
| `STATIC_CLOCK_MHZ` | 无 | `1395` | static 唯一核心变量 |
| 固定频率支持检查 | 无 | 有 | `preflight.json` 中两节点都记录 `static_clock_supported=true` |
| GPU 时钟锁定 | 无 | 有 | static 路径会按可见 GPU 范围锁频 |
| GPU 时钟 reset | 无显式锁频，无需 reset | 有 | launcher / runtime 退出路径负责 reset |
| Triton 缓存目录 | `.../triton_cache_baseline_nosave_fixenv` | `.../triton_cache_static1395_nosave_fixenv` | 避免 JIT cache 交叉污染 |
| Master Port | `30775` | `30776` | 避免两次 run 端口复用冲突 |
| 其余 workload 参数 | 相同 | 相同 | 模型、权重、数据集、拓扑、step 窗口都未改变 |

## 运行环境附加处理

| 项目 | 共同处理 | 原因 |
| --- | --- | --- |
| `DISABLE_SAVE_CHECKPOINT=1` | 是 | `sd-2` 空间紧张，避免训练结束时 checkpoint 落盘失败 |
| `/dev/shm` JIT/cache 环境链 | 是 | 避免 `CPUAdamBuilder().load()` 回退到不可写缓存目录 |
| `TORCH_EXTENSIONS_DIR` | `/dev/shm/megatron_real_qwen7b_eth_20260419/torch_extensions_tp4bit` | JIT 扩展写入共享内存 |
| `TMPDIR` | `/dev/shm/megatron_real_qwen7b_eth_20260419/tmp` | 临时文件走共享内存 |
| `PYTHONPYCACHEPREFIX` | `/dev/shm/megatron_real_qwen7b_eth_20260419/pycache` | 避免 `__pycache__` 落到受限路径 |
| `TRITON_CACHE_DIR` | baseline/static 各自独立目录 | 避免两组 run cache 污染 |
| `PYTHONDONTWRITEBYTECODE=1` | 是 | 进一步减少磁盘写入 |
| `TORCH_FORCE_NO_WEIGHTS_ONLY_LOAD=1` | 是 | 兼容当前 checkpoint 加载路径 |

## 这组表格支持的表述

- `baseline` 与 `static` 的工作负载是公平对照，主变量就是是否启用 `1395 MHz` 固定频率。
- 在真实 `Qwen2.5-7B-Instruct` checkpoint、真实数据集和 Ethernet 双机拓扑下，`static 1395 MHz` 让平均功率下降约 `30.8%`、总能耗下降约 `23.3%`。
- 代价是 `20-step` 窗口总时长增加约 `10.9%`；因此这组结果更适合作为“真实模型上的节能证据”，而不是“几乎零时长代价”的最优 headline。
- 这组结果可以和缩小版 workload 的大规模 sweep 分开陈述：前者强调真实性，后者强调频点覆盖和拓扑覆盖。
