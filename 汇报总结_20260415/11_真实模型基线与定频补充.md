# 真实模型基线与定频补充

## 结论先行

- 在 `sd-1 + sd-2` 的 Ethernet 双机环境下，同一真实 `Qwen2.5-7B-Instruct` checkpoint workload 的 `baseline + static(1005/1200/1395/1500/1650/1800/1950/2100/2250 MHz)` 十组对照现已全部完成。
- 九个固定频点相对 baseline 的结果分别是：
  - `1005 MHz`: `runtime +16.59%`、`avg power -33.26%`、`energy -22.18%`、`tokens/J +28.51%`
  - `1200 MHz`: `runtime +14.15%`、`avg power -31.84%`、`energy -22.20%`、`tokens/J +28.53%`
  - `1395 MHz`: `runtime +10.86%`、`avg power -30.78%`、`energy -23.26%`、`tokens/J +30.31%`
  - `1500 MHz`: `runtime +8.44%`、`avg power -30.28%`、`energy -24.39%`、`tokens/J +32.26%`
  - `1650 MHz`: `runtime +3.84%`、`avg power -28.73%`、`energy -25.99%`、`tokens/J +35.12%`
  - `1800 MHz`: `runtime +4.24%`、`avg power -27.69%`、`energy -24.63%`、`tokens/J +32.68%`
  - `1950 MHz`: `runtime +3.53%`、`avg power -26.60%`、`energy -24.01%`、`tokens/J +31.59%`
  - `2100 MHz`: `runtime +4.19%`、`avg power -25.94%`、`energy -22.84%`、`tokens/J +29.61%`
  - `2250 MHz`: `runtime +3.83%`、`avg power -25.09%`、`energy -22.22%`、`tokens/J +28.57%`
- 当前九点里，`1200 MHz` 基本支配 `1005 MHz`，`1500 MHz` 已经同时优于 `1395 MHz` 的时间和能耗，`1950 MHz` 给出了当前最快运行时间，但 `1650 MHz` 仍是整体更强的代表点；`2100/2250 MHz` 的加入进一步坐实了这个判断。
- 需要如实保留一个 caveat：模型/checkpoint 是真实 `Qwen2.5-7B-Instruct`，但当前 `qwen_data_text_document.{bin,idx}` 仍偏小，因此这组结果更适合表述为“真实模型 + 当前项目数据前缀”的功耗/时间证据，而不是“大规模真实语料训练”的最终证据。

## 如何解读这组结果

- 这不是缩小版 `Qwen-style` workload，而是从真实 `Qwen2.5-7B-Instruct` checkpoint 启动的 continued-pretraining / finetuning benchmarking。
- 九个 fixed-clock 点都把总能耗压低到了 baseline 的约 `74%~78%`，说明 Ethernet 慢网络环境下依然存在稳定的节能空间。
- `1200 MHz` 与 `1005 MHz` 的总能耗几乎一样，但运行时间更短，因此不再需要把 `1005 MHz` 作为主展示点。
- `1500 MHz` 已经同时优于 `1395 MHz` 的时间与总能耗，说明“更高一点但还没到默认 boost”的频点更值得保留。
- `1950 MHz` 虽然把时间代价进一步压到 `+3.53%`，但总能耗与 `tokens/J` 都不如 `1650 MHz`。
- `2100 MHz` 与 `2250 MHz` 继续证明了这一点：高频端仍然稳定，但节能收益已经明显回退，`2250` 甚至几乎没有再换来可感知的 runtime 优势。
- 当前更适合作为真实模型主展示点的仍是 `1650 MHz`：它把时间代价压到 `+3.84%`，同时把总能耗进一步降到 `-25.99%`。
- 这组结果比“只对齐结构、不加载真实 checkpoint”的 workload 证据更强，但仍需补一组“确认非占位数据集”的 rerun，才能把“真实数据集”也纳入同等级主结论。

## 后续建议

1. 在 `v100x16-1 + v100x16-2` 的 IB 环境复制同一真实模型 workload，获得跨互联环境的真实模型对照。
2. 如果还要继续加密 Ethernet 曲线，可以继续上探到 `2400+ MHz`，但当前更重要的是先保留住“`1950` 最快、`1650` 更优能耗、`2100/2250` 证明 sweet spot 没继续上移”的正式表述。

   - 二级：真实 checkpoint workload，证明方法不是只在 toy/structure-compatible 配置上成立
