# 同拓扑下基线与定频对比

## 实验目标

- 只比较**同一拓扑**下 `baseline` 与 `static fixed clock` 的差异，不把不同拓扑之间的差异混入主结论。
- 观察在 Ethernet 与 IB 两类互联环境下，固定频点是否能够在可接受的训练时间代价下显著降低总能耗。

## 共同 workload

- 模型：`36 layers / hidden 2048 / ffn 11008 / heads 16 / kv_heads 4`
- 数据集：`qwen_data_text_document`
- 训练参数：`micro_batch_size=1 / global_batch_size=4 / seq_length=2048 / train-iters=20`
- 优化与内存策略：`ZeRO-1 + CPU optimizer/offload`
- 节点规模：`2 nodes x 4 GPUs`

## 实验环境

- Ethernet：
  - 节点：`sd-1 + sd-2`
  - GPU 选择：每机 `0,1,2,3`
  - 频点：`baseline / 1005 / 1200 / 1395 MHz`
- IB：
  - 节点：`v100x16-1 + v100x16-2`
  - GPU 选择：每机 `8,9,10,11`
  - 频点：`baseline / 990 / 1080 / 1155 MHz`

## 汇总结论

- 四个同拓扑公平对照组中，固定频率都能显著降低总能耗，降幅约在 `-22.5%` 到 `-31.4%`。
- 最优频点并不总是最低频。更常见的情况是：
  - Ethernet 更偏向较高固定频点，如 `1395 MHz`
  - IB 更偏向中高频点，如 `1080 MHz` 或 `1155 MHz`
- 如果后续要做汇报中的代表性案例，当前更适合优先展示：
  - Ethernet `TP=4 / PP=2 / DP=1` 下的 `1395 MHz`
  - IB `TP=4 / PP=2 / DP=1` 下的 `1080 MHz` 或 `1155 MHz`
- 这些结果支持项目主线：预测层的价值不是直接替代训练，而是帮助用户在给定拓扑和工作负载下更快找到“合适的固定频点”。
